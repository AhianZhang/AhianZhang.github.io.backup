RestfulAPI 拦截器

Apr 2, 2018 | Spring系列 | 阅读
0 条评论

三种方式能对 RestfulAPI 进行统一处理

过滤器（Filter）

这是以组件注解的方式进行创建的，如果没有 Component 注解 也可以使用配置文件进行配置

使用 configuration 进行配置

拦截器（Interceptor)

拦截器的缺点在于每次都需要 request 和 response 遵循 J2EE 规范的，与 Spring 并不友好
拦截器比过滤器好的一点就是可以在拦截的时候获取拦截对象

单纯的写拦截器还不能够生效，需要进一步地配置

切片（Aspect)

使用 Spring 的拦截器虽然可以获得拦截的对象，但是无法获得对象的参数具体内容，此时可以使用切片
切片是一个类，对切入点使用注解的方式进行切入，注意： 在那些方法上起作用，在什么时候起作用
增强（方法）切片起作用的时候执行的业务逻辑
四个注解需要了解：
@Before ：满足切片注解的方法在调用前先执行 @Before 下的内容
@After：在执行完切片的业务逻辑后执行
@AfterThrowing：在执行切片逻辑时抛出的异常会调用
@Around：完全覆盖前三者，最常用。

@Around() 括号里面的表达式可以参见官方文档

总结

过滤器：能够拿到原始的服务器信息，但是拿不到真正请求处理的方法的信息
拦截器：既可以拿到原始的Http信息，也能拿到请处理的方法的信息，但是拿不到其值
切片：能够拿到具体处理的值，但是拿不到Http服务的信息
执行顺序：过滤器->拦截器->切片->控制器
处理异常顺序：控制器->切片->ControllerAdvice->拦截器->过滤器 一层一层的往外抛
